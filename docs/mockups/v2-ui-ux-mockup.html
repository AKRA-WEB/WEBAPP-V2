ขาด

ปุ่มหลักอยู่ล่างและมีขนาดแตะง่าย 44px+, ข้อความยาว wrap ได้, ไม่มี layout ที่ต้องเลื่อนซ้ายขวา

UX เหมือน V1, UI เสนอให้ดีกว่าเดิม

สรุปหลักคิดสำหรับ review กับผู้ใช้ก่อน implement

ต้องคงให้เหมือน V1

  • Main เป็น portal เลือกแอปตามสิทธิ์
  • Picking มีประเภทบิลเดิม: บิลจัด, บิลด่วน, บิลเรียง, จัดเตรียม
  • สร้างบิลหลายรายการในครั้งเดียว
  • มี LINE preview และ flow แจ้งกลุ่ม
  • ประวัติและเลขบิลแบบ `#NNN` ต้องจำง่าย
  • Problem report ใส่จำนวนจริงรายสินค้า

เสนอปรับ UI ให้ดีกว่า V1

  • Dashboard สรุป pending/picked/sent/problem/LINE failed
  • Action แสดงตามสิทธิ์และสถานะจริง ลดการกดผิด
  • Timeline เห็นว่าใครทำอะไรเมื่อไร
  • Mobile sticky action สำหรับงานที่ต้องกดบ่อย
  • Form label ชัด, input ใหญ่, error อยู่ใกล้ช่องที่ผิด
  • Responsive ใช้ grid/flex ที่ wrap ได้ ไม่ fix width จนล้น
ข้อเสนอนี้ยังไม่ใช่ implementation สุดท้าย ขั้นต่อไปควรให้ผู้ใช้ V1 review ว่า workflow คุ้นมือพอหรือไม่ แล้วค่อยล็อก design standard สำหรับ PR/PO/GR และโมดูลอื่น
